# GitHub Pages 白屏问题诊断

## 可能原因

### 1. Pages 配置错误（最常见）
- **现象**：线上白屏，本地 HTML 可预览
- **原因**：GitHub Pages 未从 `docs` 目录部署
- **解决**：Settings → Pages → Source: `main` 分支，Folder: **`/docs`** → 保存

### 2. type="module" 兼容性
- **现象**：部分浏览器对 inline `type="module"` 处理异常
- **原因**：ES 模块在部分环境下的执行差异
- **已做**：部署时自动改为 `type="text/javascript"`（经典脚本）

### 3. 浏览器缓存
- **现象**：修改后仍显示旧版或白屏
- **解决**：强制刷新（Cmd+Shift+R）或使用无痕模式

### 4. 部署路径
- **正确**：https://youzhaoyang.github.io/libtv-author-tag-admin/
- **验证**：若直接访问 `https://youzhaoyang.github.io/libtv-author-tag-admin/docs/index.html` 能打开，说明是 Folder 配置问题

### 5. 控制台排查
打开开发者工具（F12）→ Console：
- 若有红色报错，记录具体信息
- 若无报错，可能是 React 未挂载或渲染失败

## 验证步骤

1. 确认 Pages 配置：main + /docs
2. 等待 2–3 分钟
3. 无痕模式访问 https://youzhaoyang.github.io/libtv-author-tag-admin/
4. 检查 Console 是否有报错

## 推荐：改用 Vercel 部署

若 GitHub Pages 持续白屏，可改用 **Vercel**（免费、对 SPA 支持更好）：

1. 打开 https://vercel.com 并登录（可用 GitHub 账号）
2. **Add New** → **Project** → 选择 `libtv-author-tag-admin` 仓库
3. 保持默认，点击 **Deploy**
4. 部署完成后会得到可访问的 URL

或本地执行：`npx vercel` 按提示完成部署。
